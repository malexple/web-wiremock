# 🎭 Web WireMock

> Красивый веб-интерфейс для управления стабами [WireMock](https://wiremock.org/) —
> создан для тестировщиков и разработчиков, которым нужен полный контроль
> без ручного редактирования JSON-файлов.

![Страница стабов](./.assets/stubs.jpg)

---

## ✨ Возможности

- 📁 **Дерево стабов** — группировка по сервису и пути для быстрой навигации
- 🧙 **Мастер создания стаба** — 5 понятных шагов вместо написания JSON вручную
- ✏️ **Полноценный редактор** — запрос, ответ, query-параметры, заголовки, прокси
- ▶️ **Run Test** — отправить реальный HTTP-запрос в WireMock прямо из интерфейса и увидеть ответ
- 📋 **Журнал запросов** — все входящие запросы с заголовками, телом, таймингами и статусом совпадения
- 💾 **Профили** — сохранять, загружать, экспортировать и импортировать наборы стабов как JSON-файлы
- 🔀 **Слияние / Замена** — применить профиль поверх существующих стабов или полностью заменить
- 📥 **Импорт OpenAPI** — автоматическая генерация стабов из OpenAPI-спецификации
- 🔍 **Быстрый поиск** — фильтрация по клиенту, пути или тексту

### Профили

Сохрани текущий набор стабов как именованный профиль — восстанови одной кнопкой в любой момент.
Экспортируй профиль в JSON и передай коллеге — он загрузит его за несколько секунд через импорт.

![Профили](./.assets/profile.jpg)

| Режим | Поведение |
|-------|-----------|
| **Заменить** | Удалить все текущие стабы и загрузить профиль. Идеально для локальной разработки. |
| **Слияние** | Добавить стабы профиля к существующим. Идеально для общего k8s-окружения, где другие тестировщики уже загрузили свои стабы. |

## 🚀 Быстрый старт

### Docker Compose

```yaml
services:
  wiremock:
    image: wiremock/wiremock:latest
    ports:
      - "8888:8080"

  web-wiremock:
    image: web-wiremock:latest
    ports:
      - "8080:8080"
    environment:
      INTEGRATION_WIREMOCK_HOST: http://wiremock:8080
      INTEGRATION_PROFILES_DIR: /data/profiles
    volumes:
      - ./wiremock/profiles:/data/profiles
